Chapter 7: Software Engineering
TERM DEFINITION
Software Engineering Software engineering is the branch of computer science that deals with the design, development, testing, and maintenance of software applications. Software engineers apply engineering principles and knowledge of programming languages to build software solutions for end users.
Integrated Development Environment (IDE) IDE is a combine tools for developing software (editors, compilers, debugging tools) into a single integrated package. A prime example of such systems are those for developing applications for smartphones.
Association for Computing Machinery (ACM) It was founded in 1947 as an international scientific and educational organization dedicatednto advancing the arts, sciences, and applications of information technology.
Purpose of ACM ACM'spurpose is focusing on topics as computer architecture, artificial intelligence, biomedical computing, computers and socity, computer science education, and computer graphics, operating systems, programming language, simulation and modeling, and software engineering
Benefits of ACM Memebership Enhance your professional career or academic life with ACM member benefits that include a free subscription to ACM's flagship publication Communications of the ACM; online books, courses, videos, and webinars through the ACM Learning Center; opportunities to participate in Local Chapters, Special Interest Groups, and more.
Student ACM Membership Includes online subscriptions to Communications of the ACM and XRDS magazine, access to Online Books and Online Courses, all other benefits of ACM student membership.
Institute of Electrical and Electronics Engineers (IEEE) Improving the state of software engineering by adopting codes of professional conduct and ethics that enhance the professionalism of software developers.
Purpose of IEEE Advancing technology for the benefit of humanity
Benifits of IEEE Membership Access to technical publications Professional and educational development. Networking opportunities. Exclusive member benefits. Access to a large library of technical literature. Discounts on conference attendance, insurance programs, and IEEE products. Career alerts. Discounted pricing on insurance and other products and services.
Student IEEE Membership IEEE Student Membership connects students with professionals, academics, and the IEEE organization. It also provides access to resources and opportunities to stay current with technology.
Software Life Cycle: Development phase The development phase of the software development life cycle (SDLC) is where the software is created.
Requirements Analysis Requirements analysis is the process of determining user expectations for a new or modified product. It's also known as requirements engineering, requirements gathering, or requirements capturing.
Design Design, which is a medium-level pattern type that is used by the developers to solve problems in the design stage of development.
Implementation Software implementation is the process of integrating an application into an organization's workflow.
Testing Software testing is an investigation that provides stakeholders with information about the quality of a product or service. It also provides an objective view of the software to help businesses understand the risks of software implementation.
Modular Implementation Modularization is a software engineering technique that separates a software product's functionality into independent modules. Each module can be programmed separately and is designed to achieve a specific functionality.
Unifed Modeling Language (UML) Unified Modeling Language (UML) is a visual modeling language that helps software developers visualize and construct new systems. It's not a programming language, but a set of rules for drawing diagrams.
Use Case A use case is a written description of how users will perform tasks on your website.
Data Relationships: One-to-one, one-to-many, many-to-many In a one-to-one relationship, one record in a table is related to one and only one record in another table. In a one-to-many relationship, one record in a table can be related to multiple records in another table, but each record in the second table is related to only one record in the first table. In a many-to-many relationship, one record in a table can be related to multiple records in another table.
Design Pattern An increasingly powerful tool for software engineers is the growing collection of design. A design pattern is a predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) SQA groups which are charged with overseeing and enforcing the quality control sysytems adopted by the organization.
Software Testing Software testing is the process of assessing the functionality of a software program. The process checks for errors and gaps and whether the outcome of the application matches desired expectations before the software is installed and goes live.
Glass Box Testing Glass box testing is a software testing technique that examines the structure of a program and derives test data from the program's logic or code.
Black Box Testing Refers to tests that do not rely on knowledge of the software's interior composition.
Beta Testing Another methodology that falls within the black-box category is beta testing in which a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situitions before the final version of the product is solidified and released to the market.
Alpha Testing Similar testing performed at the developer's site is called alpha testing.
Documentation Documentation is an impportant part of a final software package and its development is an important topic in software engineering.
User Documentation The purpose of user documentation is to explain the features of the software and describe how to use them.
System Documentation The purpose of system documentation is to describe the software's internal composition so that the software can be maintained later in its life cycle.
Technical Documentation The purpose of technical documenntation is to describe how a software system should be installed and serviced.
Intellectual Property Law Intellectual property law protects the rights of creators and owners of inventions, music, writing, designs, and other works. These rights are also known as intellectual property.
Software License A software license is a document that provides legally binding guidelines for the use and distribution of software. Software licenses typically provide end users with the right to one or more copies of the software without violating copyrights.
